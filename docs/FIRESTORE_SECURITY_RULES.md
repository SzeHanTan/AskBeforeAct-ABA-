# Firestore Security Rules Setup

## Issue

The error message "Failed to get today's podcast: Failed to get podcast by date: [cloud_firestore/permission-denied] Missing or insufficient permissions" indicates that Firestore security rules need to be configured.

## Solution

Update your Firestore security rules in the Firebase Console.

### Step 1: Go to Firebase Console

1. Open [Firebase Console](https://console.firebase.google.com/)
2. Select your project
3. Go to **Firestore Database** → **Rules**

### Step 2: Update Security Rules

Replace the existing rules with the following:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // Helper function to check if user is authenticated
    function isAuthenticated() {
      return request.auth != null;
    }
    
    // Helper function to check if user owns the resource
    function isOwner(userId) {
      return isAuthenticated() && request.auth.uid == userId;
    }
    
    // Helper function to check if user is admin
    function isAdmin() {
      return isAuthenticated() && 
             request.auth.token.admin == true;
    }
    
    // ==================== USERS COLLECTION ====================
    match /users/{userId} {
      // Anyone can read user profiles (for display names, avatars)
      allow read: if isAuthenticated();
      
      // Users can create their own profile
      allow create: if isAuthenticated() && request.auth.uid == userId;
      
      // Users can update their own profile
      allow update: if isOwner(userId);
      
      // Only admins can delete users
      allow delete: if isAdmin();
    }
    
    // ==================== ANALYSES COLLECTION ====================
    match /analyses/{analysisId} {
      // Users can read their own analyses
      allow read: if isAuthenticated() && 
                     resource.data.userId == request.auth.uid;
      
      // Users can create their own analyses
      allow create: if isAuthenticated() && 
                      request.resource.data.userId == request.auth.uid;
      
      // Users can update their own analyses
      allow update: if isAuthenticated() && 
                      resource.data.userId == request.auth.uid;
      
      // Users can delete their own analyses
      allow delete: if isAuthenticated() && 
                      resource.data.userId == request.auth.uid;
    }
    
    // ==================== COMMUNITY POSTS COLLECTION ====================
    match /communityPosts/{postId} {
      // Anyone authenticated can read community posts
      allow read: if isAuthenticated();
      
      // Anyone authenticated can create posts
      allow create: if isAuthenticated() && 
                      request.resource.data.userId == request.auth.uid &&
                      request.resource.data.content.size() <= 500;
      
      // Users can update their own posts (for voting)
      // Or anyone can update for voting (upvotes/downvotes)
      allow update: if isAuthenticated();
      
      // Users can delete their own posts, or admins can delete any
      allow delete: if isAuthenticated() && 
                      (resource.data.userId == request.auth.uid || isAdmin());
    }
    
    // ==================== PODCASTS COLLECTION ====================
    match /podcasts/{podcastId} {
      // Anyone authenticated can read podcasts
      allow read: if isAuthenticated();
      
      // Anyone authenticated can create podcasts
      // (Generated by the app when users view community)
      allow create: if isAuthenticated();
      
      // Only admins can update podcasts
      allow update: if isAdmin();
      
      // Only admins can delete podcasts
      allow delete: if isAdmin();
    }
    
    // ==================== EDUCATION CONTENT COLLECTION ====================
    match /educationContent/{contentId} {
      // Anyone authenticated can read education content
      allow read: if isAuthenticated();
      
      // Only admins can create/update/delete education content
      allow write: if isAdmin();
    }
  }
}
```

### Step 3: Publish Rules

1. Click **Publish** button in the Firebase Console
2. Wait for the rules to deploy (usually takes a few seconds)

## For Development/Testing Only

If you want to allow all operations during development (NOT RECOMMENDED FOR PRODUCTION):

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

**⚠️ WARNING**: This allows any authenticated user to read/write all data. Only use during development!

## Verify Rules Are Working

After updating the rules:

1. Refresh your web app
2. Make sure you're signed in
3. Try to:
   - View community posts
   - Create a new post
   - View the daily podcast
   - Upvote/downvote posts

## Common Issues

### Issue: Still getting permission denied

**Solution**: 
- Make sure you're signed in (check Firebase Auth in console)
- Clear browser cache and reload
- Check browser console for detailed error messages
- Verify rules were published successfully

### Issue: "request.auth is null"

**Solution**:
- User is not authenticated
- Sign in through the app
- Check that Firebase Auth is properly initialized

### Issue: "Missing admin token"

**Solution**:
- Admin operations require custom claims
- Set admin claim via Firebase Admin SDK or Firebase Console
- For testing, you can temporarily allow all authenticated users

## Testing Security Rules

You can test your rules in the Firebase Console:

1. Go to **Firestore Database** → **Rules**
2. Click **Rules Playground** tab
3. Test different scenarios:
   - Authenticated user reading posts
   - Authenticated user creating posts
   - Unauthenticated user trying to read
